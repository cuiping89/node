⚙️ 运维管理

🎯 核心命令

edgeboxctl sub # 查看订阅链接与 控制面板URL edgeboxctl status # 查看所有服务及端口的健康状态 edgeboxctl logs <service> # 查看指定服务的实时日志 (Ctrl+C 退出) edgeboxctl restart # 优雅重启所有核心服务 (配置变更后使用) edgeboxctl update # 在线更新 EdgeBox 至最新版本 edgeboxctl help # 显示帮助信息
示例：
edgeboxctl logs xray

🔒 证书切换

edgeboxctl cert status # 查看当前证书类型、域名及有效期 edgeboxctl switch-to-domain <domain> # 切换为域名并申请 Let's Encrypt 证书 edgeboxctl cert renew # 手动续期 Let's Encrypt 证书 edgeboxctl switch-to-ip # 切换回 IP 模式，使用自签名证书 edgeboxctl fix-permissions # 修复证书文件的读写权限
示例：
edgeboxctl switch-to-domain my.domain.com

🌐 SNI 域名轮换

edgeboxctl sni list # 显示 SNI 域名池状态 (别名: pool) edgeboxctl sni test-all # 测试池中所有域名的可用性 edgeboxctl sni auto # 智能测试并选择最优 SNI 域名 edgeboxctl sni set <domain> # 手动强制指定一个 SNI 域名
示例：
edgeboxctl sni set www.apple.com

🔐 Reality 密钥轮换

edgeboxctl reality-status # 查看 Reality 密钥轮换的周期状态 edgeboxctl rotate-reality --force # 手动执行 Reality 密钥对轮换 (安全增强) edgeboxctl rotate-sid # 无感轮换 Reality shortId（24h宽限期）

🧬 流量特征随机化

edgeboxctl traffic status# 查看随机化系统状态和定时任务 edgeboxctl traffic reset# 重置随机化参数为默认值 edgeboxctl traffic randomize [light|medium|heavy]# 执行流量特征随机化，增强隐蔽性

示例：

edgeboxctl traffic randomize light# 轻度随机化，仅Hysteria2 仿装站点 edgeboxctl traffic randomize medium# 中度随机化，修改Hysteria2 + 协议参数 edgeboxctl traffic randomize heavy# 重度随机化，修改全协议参数

🔗 独立用户订阅URL

edgeboxctl sub show # 查看用户订阅及已绑定的设备 edgeboxctl sub issue <user> # 为指定用户下发专属订阅链接 edgeboxctl sub revoke <user> --force # 停用指定用户的订阅链接 edgeboxctl sub limit <user> <N> # 修改用户的设备上限

示例：

edgeboxctl sub revoke alice --force# 默认24小时宽限期，加--force为0宽限期 edgeboxctl sub limit alice 5

👥 网络身份配置

edgeboxctl shunt vps# VPS 直连出站（默认）
edgeboxctl shunt resi `<URL>`# 代理全量出站（仅 Xray）
edgeboxctl shunt direct-resi `<URL>`# 智能分流（白名单直连，其余走代理）
edgeboxctl shunt status# 查看当前出站模式及代理健康状态
edgeboxctl shunt whitelist {action} [domain]# 管理白名单【add|remove|reset|list】

示例：

edgeboxctl shunt direct-resi 'socks5://user:pass@host:port'
edgeboxctl shunt whitelist add netflix.com

URL格式：

http://user:pass@host:port
https://user:pass@host:port
socks5://user:pass@host:port
socks5://user:pass@host:port?sni=example.com

📊 流量与预警

edgeboxctl traffic show # 在终端查看流量使用统计 edgeboxctl alert show # 查看当前预警配置 edgeboxctl alert monthly <GiB> # 设置月度流量预算 edgeboxctl alert steps <p1,p2,...> # 设置百分比预警阈值 (逗号分隔) edgeboxctl alert telegram <token> <chat_id> # 配置 Telegram 通知渠道 edgeboxctl alert discord <webhook_url> # 配置 Discord 通知渠道 edgeboxctl alert wechat <pushplus_token> # 配置微信 PushPlus 通知渠道 edgeboxctl alert webhook <url> [format] # 配置通用 Webhook (raw|slack|discord) edgeboxctl alert test [percent] # 模拟触发预警以测试通知渠道
示例：
edgeboxctl alert monthly 1000 edgeboxctl alert steps 50,80,95 edgeboxctl alert telegram <token> <chat_id> edgeboxctl alert test 80

🧩 配置与维护

edgeboxctl dashboard passcode # 重置 Web 控制面板的访问密码 edgeboxctl alias <我的备注> # 为当前服务器设置一个易记的别名 edgeboxctl config show # 显示所有协议的 UUID、密码等详细配置 edgeboxctl config regenerate-uuid # 为所有协议重新生成 UUID 和密码 edgeboxctl backup create # 创建当前系统配置的完整备份 edgeboxctl backup list # 列出所有可用的备份文件 edgeboxctl backup restore <file> # 从指定备份文件恢复系统配置
示例：
edgeboxctl alias "香港-CN2-主力" edgeboxctl backup restore edgebox_backup_xxx.tar.gz

🔍 诊断与排障

edgeboxctl debug-ports # 检查核心端口 (80, 443) 是否被占用 edgeboxctl test # 对各协议入口进行基础连通性测试 edgeboxctl test-udp <host> <port> [seconds] # 使用 iperf3/socat 进行 UDP 连通性简测
排障流程：
→ edgeboxctl status → edgeboxctl logs xray → edgeboxctl debug-ports
